Прошу притриматись рекомендацій, щоб код був більш читаьельним і чистим:
-Iмена файлів зазвичай пишуться в нижньому регістрі.
-Важливо дотримуватися консистентності у всіх назвах файлів. Якщо ви використовуєте підкреслення для розділення слів у назвах файлів (як у filling_tables.py), слід застосовувати цей підхід у всіх файлах.
-Видалити зайві файли
-Щоб зробити структуру файлів вашого проекту більш організованою, можна розфасувати файли по папкам на основі їх призначення.
project_root/              # Коренева директорія проекту
├── data/                  # Директорія для даних
│   ├── DZ.db              # Файл бази даних
│   └── ...
├── migrations/            # SQL-скрипти для міграцій бази даних
│   ├── 001_create_tables.sql     # Створення таблиць
│   └── ...│
├── queries/               # SQL-запити
│   ├── query_1.sql        # Запит 1
│   ├── query_2.sql        # Запит 2
│   └── ...
├── scripts/               # Python скрипти для роботи з БД
│   ├── filling_tables.py  # Заповнення таблиць даними
│   └── starter_query.py   # Скрипт для стартування запитів
├── logs/                  # Логи
│   └── ...
├── tests/                 # Тести
│   └── ...│
├── .gitignore             # Файл для ігнорування в git
├── LICENSE                # Ліцензія
├── README.md              # Опис проекту
└── requirements.txt       # Залежності проекту
